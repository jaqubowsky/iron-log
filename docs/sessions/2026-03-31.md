## Co robił

Sesja teorii HTTP — 3 tematy:

1. **Cache-Control headers** — max-age, no-cache, no-store, stale-while-revalidate. Nie pamiętał max-age z homework (najważniejsza dyrektywa!). Po przeczytaniu MDN na sesji szybko zrozumiał różnicę max-age vs SWR. Praktyczny scenariusz GET /exercises — sam doszedł do max-age=3600 + SWR=86400 po naprowadzeniu (początkowo chciał max-age na miesiąc). no-cache vs no-store — poprawnie od razu
2. **ETag vs Last-Modified** — nie zrozumiał z homework ("nie mam pojęcia"). Na sesji wyjaśniony flow conditional requests (ETag → If-None-Match → 304). Różnica ETag vs Last-Modified — potrzebował naprowadzenia (mylił kierunek: myślał że data się nie zmieni a ETag tak, zamiast odwrotnie). Po korekcie zrozumiał: ETag = content-based (dokładniejszy), Last-Modified = date-based (tańszy ale niedokładny)
3. **API versioning** — URL vs header vs query param. Znał tylko URL (/v1/). Header i query param nowe. Trade-offy URL vs header — sam wymienił: URL czytelniejszy i prostszy routing, header czystsze URL ale trudniejszy routing. CDN trade-off odwrócił na explain (myślał że URL ma problem z CDN zamiast header). Po korekcie zrozumiał
